# Pack Development Workflow

1. Define the target vendor, domain, and resource type.
2. Implement or revise the adapter against the pack contract.
3. Register pack metadata and pack version.
4. Validate online and offline behavior for task binding.
5. Add or update tests for:
   adapter loading, runtime execution, and version behavior.
6. Update specification and acceptance artifacts if the supported scope changed.
7. Run real-cloud or mock validation and classify the evidence correctly.
