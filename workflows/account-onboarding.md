# Account Onboarding Workflow

1. Confirm target vendor, credential type, and expected validation level.
2. Create the account record with minimum required fields.
3. Run credential verification and capture result plus message.
4. If supported, run region and zone discovery and persist the results.
5. Register acceptance evidence:
   account code, vendor, verify result, region scope, zone scope, and limitations.
6. If the account is intended for real-cloud acceptance, verify at least one successful runtime path after onboarding.
