# SOW Acceptance Workflow

1. Extract resource and capability scope from the SOW.
2. Build a coverage matrix by vendor and resource type.
3. Check code, API, GUI, tests, and validation reports.
4. Classify each row as:
   real-cloud validated, empty-result validated, mock-only, or missing.
5. Summarize residual risks and environment-limited items separately.
6. Publish:
   coverage matrix, acceptance summary, and remaining blockers.
